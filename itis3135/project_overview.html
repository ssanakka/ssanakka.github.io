Project Overview

Project Name: CodeCollab Learning Platform

Application Purpose:

CodeCollab is an interactive web-based learning platform designed to help computer science students at UNCC collaborate on coding exercises, share resources, and enhance their programming skills through peer-to-peer learning. The platform provides a centralized space for students to practice coding, get feedback, and access learning materials.

Intended Users:
  • UNCC Computer Science students (primary users)
  • Programming instructors and TAs
  • Students learning web development
  • Peer tutors and study group members
Content Overview:
  • Interactive coding exercises and challenges
  • Student project showcase gallery
  • Resource library with tutorials and documentation
  • Study group coordination tools
  • Peer code review system
  • Programming concept explanations

Client Information

  • Client Name: Computer Science Student Association (CSSA)
  • Organization: UNCC College of Computing and Informatics
  • Client Email: cssa@uncc.edu

Page Design

1. Home Page (index.html)

  • Purpose: Introduce platform and provide quick access to main features
  • Audience: All users (new and returning)
  • Content: Welcome message, featured exercises, recent projects, quick links
  • Data Fields: Search bar, newsletter signup
  • Validation: Email format for newsletter
  • Interactive Elements: Navigation menu, search functionality, image carousel
  • Actions: Search execution, newsletter subscription, navigation to other pages

2. Exercises Page (exercises.html)

  • Purpose: Provide interactive coding challenges
  • Audience: Students practicing programming
  • Content: Coding problem statements, code editor, submission area
  • Data Fields: Code input, problem selection
  • Validation: Code syntax checking, submission limits
  • Interactive Elements: Code editor, run button, submit button, difficulty filters
  • Actions: Code execution, solution submission, filtering exercises

3. Gallery Page (gallery.html)

  • Purpose: Showcase student projects and achievements
  • Audience: Students, instructors, potential employers
  • Content: Project images, descriptions, student information, technologies used
  • Data Fields: Project submission form (read-only display)
  • Validation: Image format, text length limits
  • Interactive Elements: Image lightbox, filter by technology, search
  • Actions: Image zoom, project filtering, detailed view

4. Resources Page (resources.html)

  • Purpose: Central repository for learning materials
  • Audience: Students seeking additional learning resources
  • Content: Tutorial links, documentation, video resources, cheat sheets
  • Data Fields: Resource search, category selection
  • Validation: Search query length
  • Interactive Elements: Search bar, category filters, accordion menus
  • Actions: Resource filtering, external link navigation, content expansion

5. Study Groups Page (groups.html)

  • Purpose: Facilitate peer collaboration and study sessions
  • Audience: Students looking for study partners
  • Content: Group listings, meeting schedules, member profiles
  • Data Fields: Group search, availability calendar
  • Validation: Date/time formats, contact information
  • Interactive Elements: Calendar widget, contact forms, group filters
  • Actions: Group filtering, contact form submission, calendar navigation
